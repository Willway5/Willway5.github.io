今天学习了如何使用cocospods来替代使用第三方时拖拽核心文件的做法，把步骤写出来： 1.移除现存Ruby默认源

$gem sources --remove https://rubygems.org/

2.使用新的源

$gem sources -a https://ruby.taobao.org/

3.验证新源是否替换成功

$gem sources -l

4.安装CocoaPods

(1) sudogeminstallcocoapodsps：升级OSXELCapitan后改为sudogeminstallcocoapodsps：升级OSXELCapitan后改为sudogeminstallcocoapodsps：升级OSXELCapitan后改为sudogeminstallcocoapodsps：升级OSXELCapitan后改为sudo gem install -n /usr/local/bin cocoapods，这一点我没有用到，是看网上大神的（出自http://www.cnblogs.com/zxs-19920314/p/4985476.html?utmsource=tuicool&utmmedium=referral），暂且补充上吧，以供参考

(2) $pod setup

5.更新gem

$sudo gem update --system

新建工程，并在终端用cd指令到文件夹内 $pod search 第三方

7.新建文件 vim “Podfile”，（此处容易写错成Profile，导致反复出错而不知）

$vim Podfile

写入以下内容并保存 做法：（终端vim文件 按 i 可编辑 ，esc 退出编辑，：wq 可保存退出）

platform:ios, '6.0'

pod 'AFNetworking', '~> 2.3.1' <-------第三方

8.导入第三方库

$pod install
